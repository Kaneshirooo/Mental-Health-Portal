2.1 Technical Background
In order to guarantee dependability, scalability, and usability, the technologies, frameworks, and design principles utilized in the development of the Web-Based Guidance System for PSUSC are described in detail in this section of the document.
JavaScript was used as the primary programming language for both the front-end and back-end development, providing consistency across the entire development stack. This uniform approach reduced the learning curve for developers, simplified debugging, and accelerated the creation of system features. The back-end operated on Node.js, a runtime environment designed for real-time and event-driven applications. Node.js efficiently handled asynchronous tasks and multiple concurrent requests, making it suitable for managing numerous appointment bookings without performance issues. Express.js, a lightweight framework for Node.js, further supported server-side development by offering organized routing, middleware integration, and structured API management.
For data management, MongoDB was used as the system’s database. As a NoSQL database, MongoDB accommodated flexible, evolving, and unstructured data—an important requirement for handling user profiles, appointment records, and system logs that changed over time. Its schema-less design and ability to scale horizontally ensured fast data retrieval and stable performance as system usage increased. This allowed the system to adapt to growth without the need for major database restructuring.
RESTful APIs were implemented to manage communication between system components. These APIs followed standard HTTP protocols, making data exchange lightweight, secure, and platform-independent. Through this architecture, the system maintained smooth interactions between the client interface, server processes, and database operations. The use of RESTful APIs also supported potential integration with external services and future system enhancements.
Security was addressed through the implementation of JSON Web Tokens (JWT) for authentication and session handling. JWT provided a secure, stateless method of verifying user identities, reducing risks associated with session hijacking and unauthorized access. Its stateless nature improved scalability by eliminating the need for storing session data on the server.
The system was deployed on Vercel, a cloud hosting platform known for its reliability and automated deployment capabilities. Vercel provided continuous integration, serverless functions, and automatic scaling, ensuring that updates were deployed without interrupting system availability. This environment maintained stable system performance even during periods of high user activity.
Throughout development, Visual Studio Code functioned as the primary code editor, supplying debugging tools, syntax support, and productivity-enhancing extensions. GitHub was utilized for version control, allowing developers to track code changes, manage branches, and collaborate efficiently. Postman was used to test and validate API endpoints, enabling the team to detect and resolve issues early in the development process. By simulating real user scenarios, Postman ensured reliable API behavior and stable system performance.
Together, these technologies formed a robust and scalable system architecture that supported the needs of the Pangasinan State University San Carlos Campus Guidance Office. The selected development tools, frameworks, and deployment platform enabled efficient operation, secure data handling, and a reliable appointment scheduling experience for its users.

Their system aimed to enhance students’ decision-making by providing access to real-time employment trends and labor market indicators, thereby bridging the gap between academic preparation and career opportunities. The authors emphasized the importance of involving career counselors in the design process to tailor the system to user needs. This study is highly relevant to the current research as it demonstrates the practical benefits of incorporating dynamic, web-based tools into guidance systems. For PSUSC, implementing a similarly adaptive web-based platform could improve the institution’s ability to deliver timely and relevant career guidance, directly supporting student success in an increasingly competitive job market.

This approach ensures a systematic and internationally recognized method of evaluation, in line with the study’s objective to test the system’s usability and effectiveness.

2.3 Synthesis
The collective insights from both local and international studies highlight the indispensable role of modern guidance systems in higher education. They emphasize not only the operational challenges that institutions face but also the transformative impact of technological integration on the delivery of student services. These findings offer relevant foundations and valuable perspectives for the development of a Guidance System for Pangasinan State University – San Carlos Campus (PSU–SCC).
The reviewed literature consistently underscores the growing importance of digital transformation in guidance and counseling services, especially in academic institutions. Numerous studies (Simanjuntak & Leon, 2024; Angelina, 2022; Muhammad, 2024) highlight the inefficiencies of traditional manual systems, which hinder timely service delivery, data accuracy, and record management. This supports the need for web-based platforms that streamline processes and enhance counselor productivity.
Several studies (Nguyen et al., 2025; Zhang, 2025) explore the integration of data analytics in guidance systems. These technologies offer personalized academic and career recommendations, which are more effective than generic or group-based counseling. This implies that intelligent features should be considered in the development of modern systems to cater to individual student needs.
The theme of accessibility and usability is echoed in works by Rosmalina (2023), Fernandez & Reyes (2022), and Garcia (2020), all of which stress that intuitive design and user-friendly interfaces increase engagement and satisfaction among students. Accessibility is also critical during emergencies, as shown in studies like Bangkara et al. (2022) and Kaznosky et al. (2021), which underscore the role of remote and mobile-friendly guidance during the COVID-19 pandemic.
Mental health has also emerged as a priority in digital guidance systems. Gan et al. (2025) and Villanueva & Cruz (2023), found that digital platforms effectively support mental wellness and academic growth, advocating for integrated wellness features. However, challenges such as internet access, counselor readiness, and digital literacy remain (Santos & Delos Reyes, 2021; Mababa & Fabella, 2023), suggesting the need for infrastructure improvements and ongoing training.
The relevance of internationally recognized software quality standards is further reinforced by studies using the ISO/IEC 25010 model. For instance, Alabdulqader et al. (2021), applied this model to evaluate a university academic portal, demonstrating its effectiveness in identifying usability issues and improving user satisfaction. Likewise, Hassan and Manaf (2018), confirmed the applicability of ISO 9126—precursor to ISO/IEC 25010—in evaluating a university guidance system, validating its relevance in assessing software usability within educational environments. These works support the adoption of ISO/IEC 25010 in the present study to evaluate the system’s acceptability and performance.
Additionally, Agustin and Babaran (2021), developed a guidance records management system with SMS notifications, which streamlined communication between school administrators and parents. This underscores the significance of real-time alerts and automated documentation—features also integral to the proposed web-based guidance platform.
Finally, Philippine-based studies (Domingo, 2023; Peji, 2023; Cagayan State University, 2025) provide contextual relevance, validating that local institutions benefit from web-based systems but must address demographic diversity, policy gaps, and support for counselors.

The web interface acts as the primary medium for input and output operations, enabling seamless interaction between users and the central system. This integrated model enhances the delivery of guidance services through streamlined data management, real-time access, and user-oriented design, as supported by the principles outlined in Shikalepo’s work on educational system frameworks
